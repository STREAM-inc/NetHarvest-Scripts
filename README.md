# 🌾 NetHarvest スクリプト専用リポジトリ (NetHarvest-Scripts)

このリポジトリは、NetHarvest システムで動かす **「スクレイピングコード本体（`.py`ファイル）」** と **「いつ・どのプログラムを動かすかの設定（`sites.yml`）」** を管理するための専用リポジトリです。

あなたがローカル開発環境で作成し、テストが完了したクローラーは、このリポジトリにアップロードすることでシステムに引き継がれ、本番環境で自動的に実行されるようになります。

スクリプトの登録・同期方法には、大きく分けて**2つのやり方**があります。ご自身の経験に合わせて好きな方を選んでください。

---

## 方法A: GitHubのWeb画面から直接アップロードする（初心者向け・カンタン！）

Gitのコマンドがよくわからない方や、手っ取り早くアップロードしたい場合はブラウザから直感的に操作できます。

### 1. スクリプト（.py）のアップロード

1. ブラウザで [NetHarvest-Scripts リポジトリ](https://github.com/STREAM-inc/NetHarvest-Scripts) を開きます。
2. フォルダを辿って、スクリプトを入れたいカテゴリフォルダ（例: `sites/portal/`）を開きます。
   > ※新しくカテゴリフォルダを作りたい場合は、画面右上の `Add file` > `Create new file` から `新しいカテゴリ名/ファイル名.py` のようにスラッシュ(/)を使って入力します。
3. 画面右上の **`Add file`** ボタンから **`Upload files`** をクリックします。
4. 手元で作ったクローラーの Pythonファイル（例: `example.py`）を画面にドラッグ＆ドロップします。
5. 画面左下の **`Commit changes`** ボタンを押して保存します。

### 2. sites.yml に設定を追記する

スクリプトを置いただけではシステムは動いてくれません。次に「このスクリプトを動かしてね」と教える設定を行います。

1. リポジトリのトップ画面に戻り、**`sites.yml`** をクリックして開きます。
2. 右上にある **鉛筆マーク（Edit this file）** ✏️ をクリックします。
3. ファイルの一番下に、以下のフォーマットを参考にクローラーの設定を追記します。

```yaml
  - site_id: example             # システム用の英語ID（サイトのドメイン推奨、他と被らないように）
    name: "テストサイト"           # 日本語のサイト名（これがCSVファイルの名前になります）
    module: "portal.example"     # pythonファイルの場所（sites/以下のフォルダパス。 .py は不要）
    url: "https://example.com"   # スクレイピング対象のURL
    schedule: "0 2 * * *"        # cron式（いつ動かすか）。手動なら null にします
    enabled: true                # true なら有効、false なら無効（動かさない）
```

4. 追記できたら、画面右上（または右下）の **`Commit changes`** をクリックして保存します。
   これで全て完了です！

## 方法B: Gitコマンドで手元のPCから同期する（開発者向け）💻

VS Codeなどを使っており、Gitに慣れている方はこちらの方法がスムーズです。

### 1. リポジトリをクローンする

ターミナル（コマンドプロンプト）を開き、開発用のフォルダにこのリポジトリをダウンロードします。

```bash
git clone https://github.com/STREAM-inc/NetHarvest-Scripts.git
cd NetHarvest-Scripts
```

### 2. ファイルを配置・編集する

1. クローンしたフォルダ内の `sites/` 配下（例: `sites/portal/` など）に、作成した `〇〇.py` ファイルをコピーして配置します。
2. エディタで `sites.yml` を開き、ファイルの一番下にクローラーの設定を追記します。

```yaml
  - site_id: example
    name: "テストサイト"
    module: "portal.example"
    url: "https://example.com"
    schedule: "0 2 * * *"
    enabled: true
```

### 3. Gitでコミット＆プッシュする

設定が終わったら、以下のコマンドを順番に実行してGitHubに送信します。

```bash
# 変更されたファイルをすべて追加
git add .

# 変更内容をコミット（メッセージは適宜変更してください）
git commit -m "〇〇サイトのクローラーと実行設定を追加"

# Github（リモート）へプッシュする
git push origin main
```

## 💡 補足：cron式（スケジュール）の書き方

`sites.yml` の `schedule` に記載する式は、左から順に `分` `時` `日` `月` `曜日` を表します。よくわからない場合は以下の定番パターンをコピペしてください。

- **毎日 夜中の2時** に動かす： `"0 2 * * *"`
- **毎週 月曜の朝5時** に動かす： `"0 5 * * 1"`
- **毎月 1日の朝3時** に動かす： `"0 3 1 * *"`
- **手動でしか動かさない（定期実行しない）**： `null`

> （もっと細かく設定したい場合は、ChatGPTなどのAIに「毎週金曜の18時に動かしたいからcron式を教えて」と聞くと一発で教えてくれます！）

以上で、NetHarvest への連携は完了です！お疲れ様でした！🌾
